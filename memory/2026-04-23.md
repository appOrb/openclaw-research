### Key Learnings

1. **Transformers.js works well** for browser ML
2. **WebGPU detection** requires type casting (`navigator as any`)
3. **4 GB model download** needs good UX (progress bar)
4. **Keyword biasing** perfect for menu items
5. **Client-side ASR** viable for production

### Risks Identified

1. First-load experience (4 GB download)
2. Browser support (WebGPU only Chrome/Edge)
3. Device RAM requirements (8-16 GB)
4. Accuracy vs. Deepgram (to be tested)
